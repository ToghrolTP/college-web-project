اصول مدیریت مستقیم حافظه و ثبات‌ها

۲۵ اسفند ۱۴۰۳

این یک متن ساختگی در باب برنامه‌نویسی سطح پایین است. در این سطح از انتزاع، تمرکز اصلی بر روی مدیریت مستقیم حافظه و کار با اشاره‌گرها قرار دارد.

هر بایت از حافظه دارای یک آدرس منحصر به فرد است و برنامه‌نویس مسئولیت تخصیص (allocation) و آزادسازی (deallocation) بلاک‌های حافظه را به صورت دستی بر عهده می‌گیرد تا از نشت حافظه (memory leak) جلوگیری به عمل آورد.

دستکاری مستقیم ثبات‌های (registers) پردازنده، مانند ثبات شمارنده برنامه (Program Counter) و اشاره‌گر پشته (Stack Pointer)، امری رایج و ضروری برای بهینه‌سازی عملکرد است. 

void* ptr = malloc(sizeof(int) * 100);
if (ptr != NULL) {
    // Use allocated memory
    free(ptr);
    ptr = NULL;
}

تعامل با سخت‌افزار: زبان اسمبلی و وقفه‌ها

۱۸ اسفند ۱۴۰۳

در زبان اسمبلی، هر دستورالعمل به صورت یک به یک با یک دستور ماشین (machine code) متناظر است. این به برنامه‌نویس کنترل دقیقی بر روی سخت‌افزار می‌دهد.

اما به قیمت پیچیدگی و خوانایی پایین‌تر کد تمام می‌شود. مفاهیمی چون وقفه‌ها (interrupts)، که سیگنال‌هایی از سوی سخت‌افزار برای متوقف کردن روند عادی اجرای برنامه هستند، و همچنین دسترسی مستقیم به حافظه (DMA)، که به سخت‌افزار اجازه می‌دهد بدون دخالت پردازنده به حافظه دسترسی پیدا کند، از مباحث کلیدی در این حوزه به شمار می‌روند. 

mov eax, 1          ; System call number
mov ebx, 0          ; Exit status
int 0x80            ; Interrupt - system call

امنیت و ساختار حافظه: پشته، هیپ و آسیب‌پذیری‌ها

۱۰ اسفند ۱۴۰۳

سرریز بافر (buffer overflow) یکی از آسیب‌پذیری‌های امنیتی شایع در این سطح است که در آن، داده‌های ورودی بیش از ظرفیت بافر در حافظه نوشته شده.

و ممکن است باعث بازنویسی آدرس‌های بازگشتی و اجرای کدهای مخرب گردد. درک عمیق ساختار پشته (stack)، هیپ (heap) و نحوه فراخوانی توابع (function call conventions) برای جلوگیری از چنین خطاهایی حیاتی است.

این متن صرفاً برای نمایش ساختار و فضای مفهومی این حوزه طراحی شده و هدف آن، پر کردن فضا با محتوایی تخصصی و مرتبط با معماری کامپیوتر است. 

char buffer[10];
// Vulnerable to buffer overflow
strcpy(buffer, user_input);

// Safer approach
strncpy(buffer, user_input, sizeof(buffer) - 1);
buffer[sizeof(buffer) - 1] = '\0';
